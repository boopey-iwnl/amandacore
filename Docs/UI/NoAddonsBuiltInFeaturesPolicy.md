# No Addons, Built-In Features Only

AmandaCore does not support third-party addons.

AmandaCore may include addon-quality features as native game systems. Built-in UI features must be content-reviewed, tested, packaged, and compatible with server authority.

The UI must not provide:

- arbitrary script execution
- Lua addon execution
- runtime plugin loading
- user-installed modules
- addon API surfaces
- AddOns folder conventions

Future built-in examples may include:

- action-bar paging
- frame movement and edit mode
- bag search and sort
- objective tracker filters
- combat text filters
- threat widgets
- raid and party frames
- nameplate options
- tooltip comparison
- equipment manager
- UI profiles

These features belong in AmandaCore as reviewed, first-party systems, not as user-loaded scripts or modules.
